PDF智能问答功能

产品需求文档演示

1. 背景与目标

问题描述

传统 PDF 工具无法基于选中文本提供智能解释或对话支持，用户需手动复制粘贴并切换上下文。

目标

构建一个支持选中 PDF 内容后即时与 AI 对话的功能

响应速度 < 5s 正确率 > 95% 安全合规

需求来源

用户反馈

不懂专业文献、不便整理笔记

技术趋势

LLM 融合式阅读场景逐渐流行

内部技术研究

《AI 对话式网站构建综合研究报告》

2. 用户故事

研究人员

I want to 选中 PDF 中的复杂段落获得 AI 解读

So that 我能快速理解其含义

产品经理

I want to 对整页内容进行摘要/生成导图

So that 我能快速把握整体结构

3. 功能描述 - 正常流程

步骤 1

用户打开 PDF，浏览页面

步骤 2

选中一段文本

步骤 3

系统显示浮动菜单：AI 聊天 / 翻译 / 解释

AI 聊天 翻译 解释

步骤 4

用户点击菜单项，系统准备上下文，构造 Prompt

步骤 5

调用 LLM API（前端 fetch + 用户 API key）

步骤 6

实时返回响应内容，展示在右侧面板中

3.2 功能描述 - 异常流程

[ERR-001] 选中文本为空或过短

请选择更完整的文本片段

当用户选择空白或文本长度不足以进行有效分析时显示此提示

[ERR-002] API 请求失败

AI 服务暂时不可用，请稍后重试

当 API 请求超时或返回错误状态码时显示此提示

[ERR-003] Token 超限

选中文本过长，已自动截断

当选中文本长度超过 API token 限制时显示此提示

4. 数据

数据处理与存储

本功能主要处理用户选择的文本数据，将其发送至LLM API进行处理并返回分析结果。所有数据处理均在前端完成，不涉及后端存储。

数据流向

PDF文档

选中文本

LLM API

AI回复

数据安全

  • 用户数据不在本地存储，仅在会话期间临时处理
  • LLM API 调用使用用户自己的 API Key，保障数据安全
  • 前端直接调用 API，避免第三方服务器中转处理
